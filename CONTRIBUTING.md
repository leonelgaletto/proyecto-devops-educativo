# Contributing

Gracias por querer sumar un proyecto.

Este repo apunta a DevOps **con práctica real**. Todo debe ser simple y probado.

## Estructura obligatoria de un proyecto

Crear una carpeta nueva dentro de `projects/` con este formato:

- `README.md` (brief + pasos)
- `assets/` (material del lab: logs/configs/etc.)
- `solution.md` (solución corta y concreta)
- `scripts/` (helpers como `verify.sh`)

Ejemplo: `projects/01-linux-logs/`

## Convenciones

- El entregable debe ser claro y fácil de revisar.
- `solution.md` debe seguir el orden definido en `PRODUCT.md`.
- Evitar texto largo: bullets y puntos clave.

## Qué significa “probado”

Antes de subir un proyecto:

- Seguí el `README.md` desde cero.
- Completá el entregable.
- Verificá que la solución sea consistente con el log/material.
- Si hay scripts (`verify.sh`), que corran sin errores.

## PRs

- Un PR por proyecto o por mejora.
- Describir qué cambia y por qué.
